# 安装 Git

  1. 到 git 官网上下载，Download git, 下载后会有一个 Git Bash 的命令行工具，以后就用这个工具来使用 git

  2. 安装好后，用 git --version 来查看一下版本

# 安装 nodejs

  1. 到 nodejs 官网下载，选择 LTS 版本

  2. 安装完后，打开命令行

​ 检查一下有没有安装成功

​ 顺便说一下，windows 在 git 安装完后，就可以直接使用 git bash 来敲命令行了，不用自带的 cmd，cmd 有点难用。

# 安装 hexo

  1. 前面 git 和 nodejs 安装好后，就可以安装 hexo 了，你可以先创建一个文件夹 blog，然后 cd 到这个文件夹下（或者在这个文件夹下直接右键 git bash 打开）。

  2. 输入命令

  1. 依旧用 hexo -v 查看一下版本

    至此就全部安装完了。

  2. 接下来初始化一下 hexo

  1. 新建完成后，指定文件夹目录下有：
  • node_modules: 依赖包
  • public：存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题
  • _config.yml: 博客的配置文件
  1. 接下来去查看基础的博客

​ 打开 hexo 的服务，在浏览器输入 localhost:4000 就可以看到你生成的博客了。

​ ctrl+c 即可关闭服务。

# 创建 Gitee 仓库

  1. 注册一个 Gitee 账户

  2. 登陆 ---> 右上角 + 号 ---> 新建仓库

    • 仓库名称与路径尽量与 gitee 登录名一致
    • 仓库介绍非必填
    • 是否开源 --- 公开
    • 选择语言 ---JavaScript
    • 创建

  3. 获得仓库地址

# 部署到 Gitee 仓库上

这个时候需要先安装 deploy-git ，也就是部署的命令，这样你才能用命令部署到 Gitee

接着部署上

  • 这时候会弹出需要登录 gitee 账号的对话框，登陆即可。
  • 然后等待上传完毕

# 开启 Page 功能

  1. 打开创建的仓库
  2. 服务 --->Gitee Pages
  3. 勾选上强制使用 Https
  4. 启动
  5. 这时候就可以访问给你的网站地址了

每次重新上传到 Gitee 都需要到 Pages 去更新服务
